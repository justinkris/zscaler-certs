HOW THIS PALACE WORKS

The narrative

A boy in a Fortinet.net shirt with a Bowser-shell backpack goes on a 14-stage journey through security checkpoints, culminating at the Australian Open tennis tournament. Each scene maps to one ZIA policy-enforcement step. Scenes 7, 10, 11, and 12 go DARK — these are the POST-only steps that don't fire on plain GET requests.

How to use each scene card

Each scene has an image slot (upload your generated image), the narrative, a memory hook reminding you what it maps to, and a copyable image-generation prompt at the bottom. Click the prompt to copy, paste into your favourite image generator.

SCENES 1–6 — GET + POST
1
Red Santa & the Naughty List
Policy → Custom Malicious URLs (ATP)
GET POST
A boy in a Fortinet.net shirt and a Bowser-shell backpack visits Santa. Santa flips through a handwritten ledger and says "I don't see you on the naughty list" — waves the boy through.
HOOK
Santa's naughty list = admin-defined custom denylist of URLs. If you're on the list, you're blocked immediately. First gate in the pipeline.
2
Big Cloud & the Cascade Pale Ale
Policy → Cloud App Control
GET POST
The boy takes Santa's sleigh to a big friendly cloud. The cloud booms "Ahh the Fortinet Application — that's fine. Would you like a Cascade Pale Ale?" Boy says yes. The cloud says "Okay, you have to talk to my lil bro."
HOOK
Big cloud = Cloud App Control checks the SaaS app. Cascade Pale Ale = the "Cascading to URL Filtering" setting. By default CAC allows and URL Filtering is skipped — hence being passed to "lil bro" URL Filtering.
3
Little Cloud & the Cloud Cat
Policy → URL Filtering
GET POST
A little cloud wearing sunglasses holds a fluffy cloud-cat. The little cloud checks a clipboard and says "IT Services category — allowed. Go through."
HOOK
Little cloud = URL Filtering. Cloud-cat = URL category. Checks which category the URL belongs to and applies the admin's URL Filtering rule for that category.
4
The VIP Club Bouncer Snake
Policy → Security Exceptions (allowlist)
GET POST
The boy arrives at an exclusive VIP club. A muscular bouncer snake in an orange hi-vis vest and sunglasses checks a velvet-rope list and says "Not on the list, sorry — you don't get to skip the checks."
HOOK
VIP list = Security Exceptions allowlist. If you're on it, you skip downstream Malware, ATP, and Sandbox. The boy isn't on it, so he has to go through all the threat engines ahead.
5
The Bowser Shell & the Mario Signature
Policy → Browser Control
GET POST
A guard at the ATP (Association of Tennis Professionals) gate demands "Show me your Bowser Shell is legit." The boy chucks the shell — it transforms like a Transformer, unfolding to reveal a glowing Mario signature stamped inside.
HOOK
Bowser shell = Browser. The shell opening to reveal Mario's signature = Browser Control verifying the browser is legitimate (signed, current version, no malicious plugins).
6
"Are You Russian?" at the Aus Open
Policy → Country-Based Blocking (ATP)
GET POST
At the Australian Open, a reporter with a red microphone shoves it in the boy's face: "Are you Russian?" Boy says "Yeah I'm rushin' haha." Gets arrested instantly. Boy shouts "It's a pun m8! I'm rushin' to the court!"
HOOK
Russian / rushin' = Country-Based Blocking. ATP blocks requests to servers in suspicious countries.
SCENES 7–14 — DARK SCENES AHEAD (POST-ONLY: 7, 10, 11, 12)
7
Mal Meninga's Black Eyes
Policy → Malicious URL Block (ATP) · POST only
🌑 POST-ONLY
The scene plunges into a purple dark. Former rugby league coach Mal Meninga appears as a reporter. His eyes flash pitch black. He leans in and asks, deadly serious: "Is your Fortinet.net shirt on the global naughty list?"
HOOK — ONLY ON POST
Malicious URL Block checks the URL against Zscaler's global threat intel database (different from Santa's custom list in scene 1). Only fires on POSTs.
8
Fans Swarm for Signatures
Policy → IPS Signature Detection (ATP)
GET POST
Scene floods back to light. The boy is suddenly swarmed by cheering fans holding out paper and red sharpies, demanding his signature on page after page after page.
HOOK
Fans wanting signatures = IPS Signature Detection. Pattern-matching traffic against known-bad signatures (botnets, XSS, phishing, malicious active content).
9
The Glowing Red Book "SCP"
Policy → Suspicious Content Protection / Page Risk (ATP)
GET POST
One fan steps forward offering a bright glowing red book with the numbers "22 SCP" emblazoned on the cover. The book pulses with an ominous red light.
HOOK
SCP = Suspicious Content Protection. The book's brightness = the Page Risk score. ML-driven — catches zero-day and novel threats.
10
Pat McAfee & the Envelope Chase
Policy → Malware Protection · POST only
🌑 POST-ONLY
Scene goes dark again. A sinister envelope pops out of the glowing red book and starts chasing the boy. Pat McAfee appears out of nowhere, clutching a big file labelled "CONTRACT" ready to intercept.
HOOK — ONLY ON POST
Pat McAfee (the AV icon) = Malware Protection. AV + anti-spyware signature scanning on files being uploaded.
11
The Boy Signs the File
Policy → File Type Control · POST only
🌑 POST-ONLY
Scene stays dark. The boy grabs a pen and signs the contract file Pat holds out — checking its type, verifying its magic bytes, confirming it's the right kind of file.
HOOK — ONLY ON POST
Signing = validating. File Type Control uses magic bytes to confirm the true file type. Only fires on POST (where uploads happen).
12
The Data Shield Punt
Policy → DLP (Data Loss Prevention) · POST only
🌑 POST-ONLY
Pat McAfee produces a massive shield emblazoned with the word "DATA". He rears back and punts the sinister envelope sky-high — it bursts the dark clouds apart. Sunshine floods back.
HOOK — ONLY ON POST
Shield labelled DATA = DLP (Data Loss Prevention). Inspects POST bodies for sensitive data. Punting = blocking the outbound data leak.
13
Veigar the Crowd-Control Mage
Policy → P2P Control (ATP)
GET POST
With the scene bright again, fans suddenly swarm the boy in chaos. Veigar appears, magical staff raised, casting crowd-control spells to contain the peer-to-peer mob.
HOOK
Peer-to-peer crowd = P2P Control. Blocks P2P file sharing, anonymizers (Tor, proxy bypass), and P2P VoIP apps.
14
The Exhausted Boy
Policy → Bandwidth Control
GET POST
After the long, eventful day, the boy is utterly drained. His bandwidth is spent. He slumps into a chair, Bowser shell beside him, completely exhausted as the sun sets.
HOOK
Boy's energy spent = Bandwidth Control. The final gate — allocates how much throughput the transaction gets. Always the last step.
