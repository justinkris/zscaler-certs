HOW THIS PALACE WORKS

The narrative

Same boy as the GET + POST palace — but as a younger version. This palace maps the HTTP GET or POST Response flow — what happens when data comes back from the server. The 9 scenes cover every policy check Zscaler runs on response content before it reaches the browser.

Echoes from the GET + POST palace

Four scenes deliberately echo imagery from the GET + POST palace: the VIP snake, fans wanting signatures, the glowing SCP book, and the exhausted boy. These policies are checked on both directions of a transaction. The new scenes (4–8) are file/content engines that only make sense on the response side.
