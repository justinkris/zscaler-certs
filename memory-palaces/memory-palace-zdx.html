How this palace was built

This palace was constructed scene-by-scene through guided questioning — not handed to you ready-made. Every animal, prop, and label was either invented by you or refined together. Because your brain made the connections, the imagery is far stickier than anything generic.

Walk the building once before bed and once on waking. After two or three full walk-throughs, the rooms recall themselves whenever a ZDX exam question appears.

What this anchors
Web Probe (PDSA)Page Fetch · DNS · Server Response · Availability. No caching — every fetch is real.

Mental split: Web Probe asks "is the app fast?" Cloud Path asks "is the journey fast?" Autosense is Cloud Path for moving targets.

What this anchors
Y EngineAI-driven automated root-cause analysis — surfaces the "why" without manual digging.
ZDX CopilotNatural-language AI assistant for IT teams. Talks; Y Engine thinks.
ZDX API (PULL)Reports · Troubleshooting (start/stop Deep Trace sessions) · ServiceNow on-demand. Outside systems ask ZDX.

API vs Webhook directionAPI = outside pulls data in. Webhook = ZDX pushes events out. Opposite directions, opposite initiation.
The critical distinction: "API is a tap you open — you pull. A webhook is a pipe that sprays when triggered — it pushes."

The 60-second walk-through

Run this in your head before any ZDX question on the exam. Name the animal, name the prop, name the concept.

  1. Reception — Zscaler the Dog: Stopwatch (PFT) + battery (Availability) → ZDX Score. Wall chart with 7-day baselines per breed. Glasses with SMOOTH toggle on/off trade-off.
  2. Triage Board — Sarah's Whiteboards: User Score = worst app (Slack at 15 circled in red). Device Score = average of devices (÷3). App Tile Score = avg over period. App Score Over Time = trend across all users.
  3. Probe Lab — Three Animals: Dog (Web Probe — P-D-S-A, no cache). Kangaroo (Cloud Path — H-L-P, 3 colour-coded trails). Bloodhound (Autosense — dynamic UCaaS destination).
  4. Imaging Suite — The MRI: Deep Tracing. One user + device + app on the bed. Dial set 5–60 min. Three screens: process data, PCAP, bandwidth dumbbell. Printer outputs PDF for L2/L3.
  5. Alarm Tower — Three Sirens: Blue Application · Green Device · Red Network. Coffee-filter scope (location, group, device). Three delivery routes: Muted (portal only) · Email · Webhook (ServiceNow/Slack/PagerDuty).
  6. Records Room — Three Cabinets: Device inventory. Software installed (current + historical versions). Process running (1 min metrics, 5 min top-5 re-rank).
  7. Security Office — Keycard Printer: Full · View Only · Custom · None. Seven sections, each independently. Deep Tracing and Webhooks get their own because PCAP + external systems.
  8. AI Wing — Five Stations: Y Engine thinks (root cause). Copilot talks (natural language). API pulls (Reports/Troubleshooting/ServiceNow tap). Webhooks push (ServiceNow/Slack/PagerDuty trucks). Intune side door (endpoint analytics).
