VISUAL HIERARCHY — BROADEST TO MOST SPECIFIC
BROADEST — INFRASTRUCTURE
App Connector Group
The pool of deployed App Connectors. Defines which connectors are available. Built first — nothing else can exist without it.
↓
MIDDLE — NETWORK SCOPE
Server Group
Defines which backend servers the App Connectors in the group can reach. Links connectors to destinations. References the App Connector Group above it.
↓
MOST SPECIFIC — POLICY
Application Segment
The specific FQDNs, IP addresses, and ports that users are granted access to. This is what appears in access policies. References the Server Group above it.

KEY RULES
⟳
ANALOGY — The Warehouse
Think of ZPA like a warehouse delivery system. The App Connector Group is the whole warehouse with your fleet of trucks. The Server Group is which sections of the warehouse (aisles) those trucks are allowed into. The Application Segment is the specific product on the shelf that a customer (user) is allowed to pick up. You can't stock a shelf until you have aisles. You can't have aisles until you have a warehouse.
★
MENTAL NOTE — First-Match Policy
ZPA evaluates access policies using a first-match principle. If two policies match the same destination, ZPA picks the most specific Application Segment. Ordering matters — a broad allow rule above a narrow deny rule will grant access before the deny rule is ever evaluated. Always put more specific rules higher in the list.
★
MENTAL NOTE — Default Deny
ZPA is default deny. All application access is blocked unless explicitly allowed by an Access Policy rule. There is no implicit allow. This is the opposite of legacy VPN — you do not get access to anything just by being connected.
!
EXAM TRAP — Reversed Order
The exam frequently presents the hierarchy in reverse order (Application Segment → Server Group → App Connector Group) as a distractor. This is wrong. Always read "broadest to most specific" as infrastructure first, policy object last. If a question asks "what must be configured before an Application Segment can be created?" — the answer is Server Group (and by extension, App Connector Group beneath it).
→
EXAM TIP — Scenario Questions
Scenario: "An admin creates an Application Segment but users cannot access the app." First thing to check — is a Server Group assigned to the segment? Is the Server Group linked to an active App Connector Group? Are the App Connectors healthy and enrolled? The troubleshooting path follows the hierarchy from bottom up: Connectors → Server Group → Segment → Policy.
