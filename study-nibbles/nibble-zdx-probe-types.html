STUDY NIBBLE

ZDX Probe Types
& What They Feed

Three probe types, three scores — know which feeds which.

ZDX Digital Experience
The pattern: Each probe type measures a different layer of the user-to-app journey, and feeds a specific ZDX sub-score. Getting this mapping wrong on the exam means misidentifying what to fix and how to triage.
Probe Type Protocol What it measures Feeds
ICMP / Ping Probe ICMP Round-trip latency and packet loss to the target host Network Score
Traceroute Probe ICMP / UDP Per-hop path mapping — latency and packet loss at every hop from endpoint to app Network Score (per-hop)
Web Probe HTTP / HTTPS DNS resolution time, server response time, page fetch time, HTTP availability (200 vs 5xx) Application Score
WHAT FEEDS DEVICE SCORE?
Not probes — telemetry from ZCC
Device Score isn't fed by any probe. It comes from ZCC's direct endpoint telemetry: CPU %, memory utilisation, disk IO, battery state, OS version, and top running processes. No probe type needed — ZCC can already see everything on the device it runs on.
SYNTHETIC PROBES
Continuous · Background
Run silently on ZCC at configured frequency (e.g., every 1–5 min) for all monitored users. Build the baseline. Feed score history and alerts.
DEEP TRACE
On-demand · Per-user
Admin-triggered for a specific user + app right now. Full traceroute + packet capture + HTTP probe in one report. Use for live active issues — not trend analysis.
!
WATCH OUT — WiFi IS NOT Device Score
WiFi signal quality is measured by ZDX but it belongs to Network Score, not Device Score. The rule: Device Score = intrinsic to the hardware/OS (CPU, RAM, disk, battery). Network Score = everything from the WiFi card outward (first hop onwards).
→
EXAM TIP — TRIAGE BY SCORE
When an exam scenario describes a user with a poor ZDX experience, identify which sub-score is low:
Device low → fix the laptop (CPU, RAM, processes)
Network low → drill traceroute hops (WiFi? ISP? Zscaler edge? App edge?)
Application low → the SaaS is slow, escalate to vendor with Web Probe evidence

If the issue is happening right now and the user is live → answer is Deep Trace.
⟳
ANALOGY TO HOLD IT ALL TOGETHER
ICMP Probe = checking if the road is open (can I reach you?).
Traceroute Probe = GPS showing every turn and where you got stuck.
Web Probe = timing how long the restaurant took to serve you after you arrived.
Device telemetry = checking whether your car was the problem before you even left.
