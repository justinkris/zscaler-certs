ZPA · APP CONNECTOR · CERT/CSR · BROWSER ACCESS SEGMENT · DNS CNAME · TEST
ZPA Hard
▾
The Scenario

Safemarch's HVAC system is maintained by an external vendor who cannot install ZCC. You must give them browser-only access to the HVAC console using ZPA Browser Access — no VPN, no client software. The app will be hosted on the Zero Trust Exchange and accessed via a public URL.

6-Stage Configuration Flow
  • 1
    Provision the App Connector — copy the OnPrem provisioning key from ZPA, transfer via WinSCP to 10.0.0.3, activate. (Same flow as EDU-200 Lab 6 — key must be named exactly provision_key)
  • 2
    Create a Web Server Certificate (CSR) — generate in ZPA, sign with the enterprise CA on Corp: Win Svr, upload the signed cert back to ZPA
  • 3
    Create the HVAC Application Segment — with Browser Access enabled, bind the cert, set protocol/port and segment/server groups
  • 4
    Create the Access Policy rule — Allow HVAC, with two criteria: App Segment = HVAC AND Client Type = Web Browser
  • 5
    Create the DNS CNAME record — on Remote: DNS Server, in zone hvac.safemarch.com, pointing your StudentID alias to the ZPA Canonical Name
  • 6
    Test from Remote: Linux Client PC — no ZCC, browser only, authenticate as hvac@<StudentFQDN>

  • Task 3.1: Provision the Corp: ZPA Connector — start SSH, WinSCP upload to /home/admin/, stop SSH, cp to /opt/zscaler/var/, start service, enable on boot, verify active in ZPA portal
  • Task 3.2: In ZPA → Configuration & Control → Certificate Management → Certificates → Create CSR with HVAC details above
  • Task 3.2: Download the CSR, take it to http://host-1.patraining.safemarch.com/certsrv, submit as Advanced Certificate Request, template: Web Server, download as Base 64
  • Task 3.2: Upload the signed certnew.cer back to ZPA — the Expiry Date should now show and the Upload button disappears
  • Task 3.3: Add the HVAC Application Segment with Browser Access checkbox ticked — all settings from the naming table above

⚠

The Client Type = Web Browser criteria is critical. Without it, the policy won't match Browser Access connections — only ZCC clients match without it.

🔑

When pasting the Canonical Name into DNS Manager, paste into Notepad first to strip out the word "CNAME" if it appears at the start. Only the value (12345.abc.zpa-app.net.) goes in the target field.

📍

The Canonical Name value lives at: ZPA → Resource Management → Application Management → Browser Access → click the HVAC app name to expand.

💡

Browser Access serves HTTPS to the client regardless. Port 80/HTTP here refers to the connection between the App Connector and the backend application server — not the client-facing connection.

📘

NIST CSF: A risk management framework (not an attack taxonomy). Maps Zscaler's protections to NIST functions: Identify, Protect, Detect, Respond, Recover. Used for compliance posture reporting.

Apply Cyberthreat Protection Services Requirements

ZIA · SSL INSPECTION · BANKING BYPASS · CUSTOM URL CATEGORY
ZIA Capstone
▾
Configuration Requirements

Configure the SSL inspection policy to meet Safemarch's privacy requirement: do not inspect user access to personal banking accounts. All other HTTPS traffic continues to be inspected.

  • Create a new SSL Inspection rule (or modify existing) to set action Do Not Inspect for banking URLs
  • Use either a custom URL category (e.g. Banking Exclusions) or the predefined Finance URL category as the criteria
  • Ensure the SSL bypass rule sits above the catch-all Inspect rule in the policy order
Testing Requirements
  • Visit https://www.icicibank.com → certificate is issued by the real bank CA (not Zscaler) — bypass is working
  • Visit a non-banking HTTPS site → certificate is still issued by Zscaler — inspection is still active for other traffic
Configuration Hints
💡

SSL inspection rule order matters. Do Not Inspect bypass rules must have a lower order number (higher priority) than the catch-all Inspect rule. Order 1 = evaluated first.

⚠

Make sure any existing SSL rules don't already cover banking — check before adding a new one. Duplicate conflicting rules can produce unexpected results.
